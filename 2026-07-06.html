MCP 2026-07-28 协议重设计：彻底移出会话状态、引入 MCP Apps、弃用 Roots/Sampling/Logging

MCP 2026-07-28 规范发布候选版本（5-21 由 David Soria Parra 与 Den Delimarsky 公布）/ 核心改动：协议彻底无状态化——每个请求自包含，无需初始化握手/无 session ID/可被 round-robin LB 直接路由 / 新增 Extensions 框架 + MCP Apps（server 渲染 UI）/ Tasks 升级为 Extension / 授权模型对齐 OAuth 2.0 + OIDC / 弃用 Roots/Sampling/Logging

MCP 无状态化是协议自发布以来最大的一次重设计。核心影响：以前每个 MCP server 必须维护 session 状态、强制 sticky session、深包检测网关——这对自托管是噩梦、对 Serverless 几乎不可行。无状态后，MCP server 可以跑在 Cloudflare Workers/Vercel Edge/Lambda 任意 LB 后面，部署成本和工程负担都断崖式下降。MCP Apps（server 渲染 UI）让 MCP 从"工具数据协议"升级为"应用层协议"——这是 Anthropic 给 MCP 生态留下的最大想象空间，也是对 OpenAI 同期推 Apps SDK 的正面回应。7-28 正式发版前，所有 server 实现都该开始测 RC 了。
